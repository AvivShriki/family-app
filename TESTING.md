# רשימת בדיקות לפני פריסה — family-app

מתוחזק על ידי מור (QA). כל באג שקורה פעמיים מקבל כאן שורה קבועה.
פריסה תמיד דרך `scripts/deploy.sh` — הוא אוכף את השערים האוטומטיים לבד.

## שערים אוטומטיים (בתוך scripts/deploy.sh)

- [ ] `DEMO_MODE = false` — פעם שנשכח דלוק, ההתחברות נעלמה מהאתר החי
- [ ] `npm test` עובר (חישוב גיל, משכי זמן, אמוג'י מוצרים)
- [ ] `npx tsc --noEmit` נקי
- [ ] נתיבי `/assets/` ו-`/_expo` תוקנו בבאנדל (תת-תיקיית `/family-app`)
- [ ] `.nojekyll` קיים ב-gh-pages — פעם שנמחק, האתר עלה בלי JS

## בדיקות ידניות בדפדפן (מצב דמו, רוחב 375)

- [ ] התחברות: מסך הכניסה נטען בלי שגיאות קונסול
- [ ] בית: שלוש השורות עם סטטוס חי
- [ ] לוח שנה: לחיצה על יום מסננת; "+" ממלא את התאריך הנבחר; שמירת אירוע
- [ ] קניות: הוספת פריט, סימון, "נקה מסומנים"
- [ ] מעקב תינוק: הוספת ארוחה עם ויטמין D + ברזל יוצרת 3 רשומות באותה שעה
- [ ] מעקב תינוק: עריכת רשומה מעדכנת בלי ליצור כפילות
- [ ] הגדרות תינוקת: שינוי תאריך לידה מעדכן את הגיל בכל המסכים
- [ ] אין גלילה אופקית באף מסך (scrollWidth == clientWidth)

## תסריט מכשיר אמיתי (אביב מריץ אחרי פריסות גדולות)

1. לפתוח את האתר בספארי באייפון (לא בהדמיה)
2. לוודא שכפתורי התחתית לא מוסתרים על ידי פס המחוות
3. לגלול הצידה בכוח בכל מסך — לוודא שלא נחשף רקע לבן
4. להוסיף רשומת ארוחה אמיתית ולוודא שנוי רואה אותה במכשיר שלה
